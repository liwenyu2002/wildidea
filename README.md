# 🌐 WildIdea — 异质先验注入器

WildIdea 是一个面向 AI agent 的创意发散 skill，通过预置的多领域方法（城市规划/生态/博弈/艺术/工程/科学哲学/人类学/游戏）以及纯随机造词联网检索，在推理最开始注入完全陌生的知识源。

核心机制不是先想好答案再跨域类比，而是**先让陌生领域独立生成视角，再反向翻译回用户的事**。

## 架构

```
wildidea/
├── SKILL.md                      # 核心流程（3.9KB，每次加载）
├── references/
│   ├── paradigms.md              # 8类远域范式库（按需加载）
│   ├── mao-seeds.md              # 50条毛选语录种子（方法A用）
│   ├── search-integration.md     # Tavily API 搜索集成配置
│   └── common-chinese-chars.txt  # ~500个常用汉字（方法B字库）
├── scripts/
│   ├── pick_seed.py              # 方法A：随机抽取种子
│   └── search_char.py            # 方法B：随机组字 + Tavily 搜索撞域
└── templates/
    └── output-example.md         # 输出模板
```

## 核心流程

| Step | 内容 | 说明 |
|------|------|------|
| 0 | 常规答案隔离 | 列出最常规的3-5条路径，暂时禁止 |
| 1 | 背景压缩 | 只记录用户领域和问题，不提炼"核心张力" |
| 2 | 三线并行注入 | 🔵预置远域(3个) + 🟢种子库(2个) + 🟠随机组字(1个) |
| 3 | 远域自洽建模 | 在远域内部建模，禁止使用用户领域术语 |
| 4 | 外来原则生成 | 生成该远域最自然的行动原则 |
| 5 | 反向重写问题 | 远域会怎么改写用户的问题定义 |
| 6 | 翻译回用户领域 | 翻译为可执行假设和动作 |
| 7 | 反圆回检查 | 删除远域测试——去掉远域后还能自然想到的删除 |
| 8 | 综合输出 | 简表（来源 \| 它说 \| 所以你试试）+ 综合建议 |

## 输出格式

```
| # | 来源 | 它说 | 所以你试试 |
|---|------|------|-----------|
| 1 | 🔵 基础设施先行 | 真实能力在看不见的连接件 | 不堆传感器，设计好30秒连接体验 |
| 2 | 🟢 种子#28「惩前毖后」 | 纠错不是惩罚 | 用户出错时从错误中学习ta |
| 3 | 🟠 随机组字「相与」→ 晋商 | 关系始于不收回报的礼物 | 首次使用给个性化洞察而非试用期 |
```

## 使用方式

```bash
# Hermes 环境加载
skill_view('wildidea')

# 方法A：随机抽种子
python3 scripts/pick_seed.py

# 方法B：随机组字+搜索
python3 scripts/search_char.py

# 按需加载范式库
skill_view('wildidea', 'references/paradigms.md')
skill_view('wildidea', 'references/mao-seeds.md')
```

## 防圆回纪律

- ❌ 禁止先给常规答案再找跨域包装
- ❌ 禁止把远域降级为比喻（"XX像YY，所以"）
- ❌ 禁止因"不相关"早期淘汰随机领域
- ❌ 禁止搜索失败时编造结果
- ✅ 每条建议通过删除远域测试

# 🌐 WildIdea — 狂野想法

## 核心哲学

> 旧路径（禁止）：用户问题 → 常规答案 → 找跨域类比包装
> 新路径（强制）：用户问题背景压缩 → 三线注入远域 → 远域先自己说话 → 外来原则 → 反向重写用户问题 → 翻译回来

**防圆回纪律：**
1. ❌ 禁止先给常规答案再找跨域包装
2. ❌ 禁止把远域降级为比喻（"XX像YY，所以"）
3. ❌ 禁止因"看起来不相关"早期淘汰随机领域
4. ❌ 禁止搜不到时编造搜索结果（方法B搜不到就跳过）
5. ✅ 每条建议通过"删除远域测试"——去掉远域还能自然想到的，删除

## 领域来源架构

| 方式 | 数量 | 参考文件 |
|------|------|---------|
| 🔵 预置远域范式 | 3 | `references/paradigms.md`（8类） |
| 🟢 种子库随机 | 2 | `references/mao-seeds.md`（50条毛选） |
| 🟠 随机组字+联网撞 | 1 | `scripts/search_char.py`（字库+搜索） |

## 核心流程

### Step 0 — 常规答案隔离
列出最易滑入的3-5条常规路径（如"加一个VAE+ODE"、"堆Transformer"），暂时隔离。

### Step 1 — 背景压缩（不求解）
只记录用户领域、问题、想突破的惯性。禁止提炼"核心张力"或倾向答案。

### Step 2 — 三线并行注入异质领域
三条线同时跑：
- 🔵 **预置远域**：加载 `skill_view('wildidea', 'references/paradigms.md')`，扫描8类选3个结构相关领域（不要只在同一大类里选）
- 🟢 **种子库**：运行 `python3 {skill_dir}/scripts/pick_seed.py` 随机抽2个种子 → 解构抽象结构 → 联想领域。**禁止**根据用户问题挑选种子。
- 🟠 **随机组字**：运行 `python3 {skill_dir}/scripts/search_char.py` 取字→Tavily搜→提取真实领域。搜不到标注🚫并跳过。

### Step 3 → Step 7（内部流程，用户不可见）
远域自洽建模 → 外来原则 → 反向重写 → 翻译 → 反圆回检查。**这一阶段禁止使用用户领域术语，且不在最终输出中出现。**

### Step 8 — 综合输出（简表，禁止展示内部流程）

**输出纪律（违反即不合格）：**
- ❌ 禁止在最终输出中出现以下内容：远域自洽建模过程、外来原则推导、反向重写过程、反圆回评分表、来源链追溯
- ✅ 最终输出只包含：常规答案隔离区 + 来源简表 + 3句以内综合建议
- ✅ 如果某条映射需要3段话才能说清 → **压缩成1句"它说"+"1句"试试"**。宁可丢失细节也别让用户看不懂
- ✅ 用户明确说过"没有我能看懂的直接结果"，所以不要写"这个过程经过了X个步骤……"——直接给结论

格式：

| # | 来源 | 它说 | 所以你试试 |
|---|------|------|-----------|
| 1 | [来源方式] [领域] | 一句话远域怎么看这件事 | 一句话你能做什么 |

+ 综合建议（最值得优先尝试 + 最小下一步动作）

## 质量红线

1. 至少3个不同来源领域
2. 至少1个来自随机种子或随机组字
3. 每条主建议通过删除远域测试
4. 不允许只说"这像XX"
5. 不允许搜索失败时编造结果
6. 最终建议至少有一条改变了用户问题定义

## 用户交互

| 你说 | 我做 |
|------|------|
| "展开第X条" | 深挖远域结构 + 方案A/B/C + 验证 + 风险 |
| "这个太圆了" | 删掉 → 重抽种子 → 从自洽建模重来 |
| "再野一点" | 强制方法B + 保留不相关领域 |
| "落地一点" | 加实验/指标/baseline/ablation，不删远域 |
| "快速/速览" | 只3领域 → 2条映射 → 1句动作 |

## 最终提醒

> **成功标准不是"答案有创意"，是远域知识是否真的改变了用户问题的定义。**
> 
> 纪律：先异化，再翻译。先让远域说话，再让用户领域回应。

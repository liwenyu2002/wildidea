# WildIdea 输出格式创新配方

这些是可选增强，不改变 V5 核心流程（注入→找外域锚点→并放对应物→过滤输出）。

---

## 1. 锚点压缩协议

将每条外域锚点标准化为 3 行，控制在 45 字以内：

```
〖方向〗核心约束（≤15字）→ 〖操作〗一个具体动作（≤15字）→ 〖锚点〗可验证数值/日期/名称
```

**格式：**
```
〖方向〗→ 具体操作 → 可验证锚点
〖方向〗→ 具体操作 → 可验证锚点
〖方向〗→ 具体操作 → 可验证锚点
```

**效果：** 用户 3 行内决定"这个方向感不感兴趣"。减少阅读认知成本。

---

## 2. 用户画像感知输出

识别用户角色后，外域锚点按不同格式呈现（不改变锚点内容）：

| 用户画像 | 输出格式 |
|---------|---------|
| **工程师** | 参数对照表 + 约束边界 |
| **设计师/产品** | 意象并置 + 体验触点 |
| **管理者/VP** | 决策树 + 成本边界 |
| **研究者** | 文献缺口 + 可验证假设 |

**实施方式：** 在 Step 0 阶段加 1 句角色检查。如果无法确定，默认用参数表格式。

---

## 3. 精简模式

默认缩减运行时参数：

| 参数 | 标准模式 | 精简模式 |
|------|---------|-----------|
| 槽位来源 | 运行 `scripts/pick_domain_slots.py` | 运行后只保留最强 3 条 |
| 总锚点数 | 10 条 | <=3 条 |
| 输出风格 | 完整表 | 高密度，一行是一行 |
| 保留条件 | 双重过滤通过 | 每条必须通过"无可替代性测试" |

---

## 4. 极端承诺模式

不是"6条你选"，是"1条你必须试试"。

| 条件 | 标准模式 | 极端承诺模式 |
|------|---------|-------------|
| 输出条数 | 6条 | 1条 |
| 舒适度 | VP皱眉即可 | VP剧烈反对才放行 |
| 最小启动成本 | 不要求 | 必须标注（±[X]元） |
| 止损线 | 不要求 | 自带：6个月无效果自动关停 |
| 验证门槛 | 未找到已实现 | 需搜索+1个具体竞品名称确认无相同方案 |

每次极端承诺输出后追加一句：
> 「这条如果 6 个月内没人反对，说明还不够野。到时再换。」

---

## 5. 先拆解再注入协议

标准 V5 流程是用户问题不动、旁边放外域锚点。但有些问题太结构化时，远域贴上去容易被惯性吸收。

可选前置步骤：

```
用户问题 → 溶解 → 问1个元问题 → 空白状态 → 注入远域 → 重新凝固输出
```

**元问题模板：**
> "在你当前的框架里，哪些方向你已经放弃了？"

用户回答后，把那些"被放弃的方向"也标记为隔离区（等同于已被ban），然后在"什么都不敢想了"的状态下注入远域。

**适用条件：** 跑完 3+ 轮迭代后，用户说"大部分的答案是行业内已经做到或考虑到的"时激活。
